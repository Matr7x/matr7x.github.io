团队协作神器Phabricator

2015-10-09

Phabricator简介

Phabricator是一个开源软件开发协作平台。

Phabricator早期是facebook内部使用的工具，后来phabricator的首席工程师离开了facebook创业并开源了Phabricator。

Phabricator的功能非常多，很多功能我也没有熟悉，今天主要介绍如下几个主要功能：

  1. Code Review
  2. Projects/Tasks and Bugs管理
  3. 代码仓库
  4. Wiki

Phabricator官方文档

地址：Phabricator User Documentation

Phabricator主要功能及使用方法

1. Code Review

Code Review是Phabricator提供的最主要的功能之一，也是技术人员最常用的一个工具。

在Phabricator下使用Code Review需要安装一个命令行工具：Arcanist

2. 项目管理

Phabricator可以以项目来组织协同工作，项目中可以指定参加的用户，可以给用户指定和分配Task或者Bug。

3. 代码仓库

Phabricator本身是自带代码仓库功能的，可以直接把Phabricator作为Git服务器使用。由于我们公司主要使用GitLab，因此Phabricator没有配置自身的代码仓库功能。

我们可以把GitLab的代码仓库导入到Phabricator中，这样远端Git的代码和Commit信息我们就都能看到了，同时可以和Project/Task进行关联。

4. Wiki

Wiki是我们最常用的功能之一，支持Markdown语法，撰写文档非常方便。相对于GitLab的Wiki的好处在于其文档的层次结构，用户可以很容易找到自己想要的文档。同时Wiki可以和Phabricator的一些对象进行关联，比如Task，Repository，文件，图片等。详细的文档请参考：Remarkup Reference。同时相对于Markdown，Phabricator的Wiki提供了更加丰富的表现形式。
